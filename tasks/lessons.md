# Lessons

- CSV 批处理需求里，若用户要求“对应的 csv 文件”，优先理解为写回原 CSV；不要默认生成新结果文件。
- API Base 这类全局参数优先放节点输入，只有用户明确要求逐行差异时才放 CSV 列。
- demo 工作流要体现用户指定的连接方式；用户说外部文本节点接入，就必须用文本节点连入对应参数。
- 用户明确要求不要修改原表时，输入表必须只读；状态、失败原因、保存路径、文件名写入独立结果表。
- 批量表格只放逐行变化的内容；format、quality、background、moderation 这类共同生成参数放节点输入。
- GPT Image 2 批量节点并发数默认 10，最大允许 999。
- 批量节点允许高并发时，必须同时做运行时数值校验、实际 worker 保护、输出路径边界校验、下载大小限制和结果文件原子命名。
- Omni 图片 URL 参数应按用户指定拆成 `image_1_url`、`image_2_url`、`image_3_url` 三个独立链接字段；不要把第三张图默认做成多 URL 文本框。
- 视频上传要先做本地大小预检，阈值按用户确认值走，不要只等服务器返回 413。
